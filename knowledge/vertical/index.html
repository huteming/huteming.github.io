# 单行文字垂直居中

设置 height = line-height

提示

很常用，主要用于文字的排版，也可以用于图片元素居中

.outer {
  height: 100px;
  background: lightblue;
  line-height: 100px;
}

# 块级元素

# flex 布局

提示

优点：更灵活，也更简洁，可维护性也更强。只要不考虑 IE，这个方案几乎是最优选择吧。

缺点：如果还在使用 IE 浏览器的话，flex 布局就没那么香了

.outer {
  height: 100px;
  background: lightblue;
  display: flex;
  align-items: center;
}

# 绝对定位 + margin: auto

提示

优点：不需要提前知道尺寸，兼容性好

缺点：目前已经不建议使用绝对定位 absolut 了，如果还在用 IE 开发，这个方法还是比较推荐的。

.outer {
  height: 100px;
  background: lightblue;
  position: relative;
}

.inner {
  position: absolute;
  top: 0;
  bottom: 0;
  left: 0;
  right: 0;
  height: 50px;
  margin: auto;
}

# display: table-cell

提示

优点：也是一个比较好的实现方式，较简洁

.outer {
  height: 100px;
  background: lightblue;
  display: table;
  width: 100%;
}

.inner {
  display: table-cell;
  vertical-align: middle;
}

# 绝对定位 + margin

提示

优点： 兼容性不错，也算是一点小技巧吧。

缺点：需要提前知道 child 的尺寸，margin-top: -(高度的一半);现在已经不建议使用绝对定位 absolute，特别是在移动设备上。

.outer {
  height: 100px;
  background: lightblue;
  position: relative;
}

.inner {
  position: absolute;
  top: 50%;
  left: 0;
  width: 150px;
  height: 50px;
  margin-top: -25px;
}

# 绝对定位 + translate

提示

优点：不需要提前知道尺寸

.outer {
  height: 100px;
  background: lightblue;
  position: relative;
}

.inner {
  position: absolute;
  top: 50%;
  left: 0;
  height: 50px;
  transform: translate(0, -50%);
}

# padding

提示

缺点：如果高度固定，需要提前计算尺寸（只在某些特定情况适用）。

.outer {
  height: 100px;
  background: lightblue;
  padding: 20px;
  box-sizing: border-box;
}

.inner {
  height: 60px;
}

# 伪元素

}

# calc

提示

缺点：需要计算，在特定场景下可能比较合适。

百分比参照物

height: 父级的 height 值

width: 父级的 width 值

margin: 父级的 width 值

padding: 父级的 width 值

.outer {
  height: 100px;
  background: lightblue;
}

.inner {
  position: relative;
  background: lightgreen;
  height: 20px;
  float: left;
  top: calc(50% - 20px / 2);
}
